# Электронное меню «Ресторан-кафе Стамбул»

Лёгкий демонстрационный прототип электронного меню на чистых HTML, CSS и JavaScript. Клиент выбирает блюда, проверяет корзину, указывает тип заказа и открывает WhatsApp с готовым текстом для менеджера.

В проекте нет backend, сборщика, внешних библиотек, CDN, Google Fonts и удалённых изображений.

## Как открыть

Для простого просмотра откройте `index.html` двойным щелчком. Для наиболее корректной локальной проверки можно запустить HTTP-сервер из папки проекта:

```powershell
python -m http.server 8000
```

После этого откройте `http://localhost:8000/`.

## Как показать демо клиенту

1. Откройте основной файл `index.html` или запустите локальный HTTP-сервер командой выше.
2. Покажите выбор категории и добавление блюд кнопками плюс/минус.
3. Откройте нижнюю корзину, укажите номер для связи и выберите предзаказ, доставку или самовывоз.
4. Для доставки укажите тестовый адрес и покажите готовый текст заказа в WhatsApp.
5. Не нажимайте финальную кнопку «Отправить» в WhatsApp, если это только демонстрация.

`preview.html` — старый исходный макет и в текущем приложении не используется. Точка входа проекта — `index.html`.

## Где менять настройки

Основные настройки находятся в объекте `CONFIG` в начале `app.js`:

- `whatsappPhone` — WhatsApp-номер без плюса;
- `restaurantName` — название ресторана;
- `cuisine` — название кухни;
- `schedule` — график;
- `currencyText` — валюта.

Блюда находятся в массиве `MENU` в `app.js`. В каждой позиции можно изменить название, описание, цену, категорию и путь к изображению. Категории формируются автоматически из поля `category`.

Логотип: `images/logo-stambul.png`.

Изображения блюд находятся в `images/dishes/`:

- `mercimek-corbasi.webp`
- `iskender-kebab.webp`
- `adana-kebab.webp`
- `pide-cheese-meat.webp`
- `lahmacun.webp`
- `dolma.webp`
- `meze-assorti.webp`
- `choban-salad.webp`
- `baklava.webp`
- `sutlac.webp`
- `turkish-tea.webp`
- `ayran.webp`

Фотографии приведены к единому формату WebP 1000×750 px. Чтобы заменить фотографию, сохраните локальный файл в эту папку и измените поле `image` нужного блюда в `MENU`.

Текущие интернет-фотографии используются временно для демонстрации. Перед коммерческим запуском их лучше заменить на реальные фотографии блюд ресторана с подтверждёнными правами использования.

## Номер через ссылку

Номер менеджера можно временно переопределить query-параметром:

```text
index.html?phone=79991234567
```

Номер очищается от плюса, пробелов, скобок и дефисов. В интерфейсе он не отображается.

## Самопроверка

Откройте:

```text
index.html?test=1
```

Результат 64 встроенных тестов появится в консоли браузера и небольшом блоке на странице. Параметры можно объединять: `?phone=79991234567&test=1`.

## Как работает WhatsApp

Кнопка создаёт ссылку `wa.me` с URL-кодированным русским текстом заказа. Браузер открывает приложение или сайт WhatsApp, но сообщение не отправляется автоматически: это ограничение и защитный механизм WhatsApp. Клиент сам проверяет текст и нажимает «Отправить».

В оформлении заказа есть обязательное поле «Номер для связи». Поддерживаются российские и международные номера, например: `89991234567`, `+79991234567`, `+380501234567`, `+447911123456`, `+14155550123` и `+60123456789`. Российские номера с `8` автоматически приводятся к `+7` и после ухода из поля показываются как `+7 (999) 123-45-67`. Международные номера сохраняются в компактном формате `+кодСтраныНомер`. В WhatsApp номер добавляется как обычный текст, а не как `wa.me`-ссылка: заказ всегда отправляется на номер менеджера из `CONFIG.whatsappPhone`.

Быстрые нажатия на плюс/минус оптимизированы для мобильных браузеров: кнопки обрабатывают касание без задержки и не запускают полную перерисовку меню.

## Сохранение корзины

Корзина, выбранный тип заказа и поля формы, включая номер для связи, сохраняются в `localStorage` под ключом `stambul-menu-cart-v1`. После перезагрузки страницы данные восстанавливаются. Кнопка «Очистить» удаляет позиции корзины и номер для связи; остальные поля формы остаются заполненными.

## Подготовка реальных фотографий

- Не используйте оригиналы с телефона размером 5–10 MB.
- Подготовьте фото шириной 800–1000 px.
- Рекомендуемый формат — WebP.
- Желательный размер одного фото — до 150–250 KB.
- Все фото должны храниться локально в проекте.
- Карточки уже используют `object-fit: cover`.

## Ограничения демо

Заказ не записывается в базу данных, наличие блюд не синхронизируется, оплата на сайте отсутствует. Итоговый заказ считается на устройстве клиента и передаётся менеджеру только через WhatsApp.

## Публикация на GitHub Pages

1. Создайте новый репозиторий на GitHub.
2. В папке проекта выполните:

   ```bash
   git init
   git add .
   git commit -m "Публикация электронного меню"
   git branch -M main
   ```

3. Добавьте remote, подставив адрес своего репозитория:

   ```bash
   git remote add origin https://github.com/USERNAME/REPOSITORY.git
   ```

4. Отправьте проект в GitHub:

   ```bash
   git push -u origin main
   ```

5. На странице репозитория откройте **Settings → Pages**.
6. В поле **Source** выберите **Deploy from a branch**.
7. Выберите ветку **main** и папку **/root**.
8. Нажмите **Save**.
9. Дождитесь публикации и откройте выданную GitHub Pages ссылку.

GitHub Pages публикует проект как статический сайт. Главная точка входа — корневой `index.html`; серверная часть и сборка не требуются.
